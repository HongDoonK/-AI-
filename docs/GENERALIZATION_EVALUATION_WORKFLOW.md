# 추천 일반화 성능 평가 워크플로우

## 목적

자산 정책 하나에 과적합하지 않고 문화, 주거, 창업, 금융 세부 의도에서도 추천 품질이 유지되는지 검증한다. 추천 수정은 특정 문장 스냅샷이 아니라 도메인 충돌 상황을 재현하는 테스트로 고정한다.

## 평가 범위

- 조건 추출: 지역, 시/군/구, 나이, 관심 분야, 주거 상태, 고용 상태
- 추천 랭킹: 금융 자산형성, 금융 대출, 문화생활, 월세/전세 주거, 창업
- 실제 데이터 확인: `data/youth_policy.db` 기준 상위 5개 결과
- 신청 도움: 추천된 정책의 신청 채널, 필요 정보, 기본 서류 fallback

## 수정 워크플로우

1. 조건 추출 회귀 테스트를 먼저 작성한다.
   - 대상 파일: `tests/test_condition_extractor.py`
   - 예: `문화생활`은 `복지`가 아니라 `문화`, `생활비 대출`은 `복지`가 아니라 `금융`

2. 도메인 충돌 synthetic 테스트를 작성한다.
   - 대상 파일: `tests/test_retriever_ranking.py`
   - 한 테스트는 한 가지 충돌만 검증한다.
   - 지역 적합도가 높은 일반 항목과 task 적합도가 높은 전국 항목을 같이 넣어 정렬 기준을 검증한다.

3. RED를 확인한다.
   - 실행: `python -m unittest tests.test_condition_extractor tests.test_retriever_ranking`
   - 실패가 기대한 assertion인지 확인한 뒤에만 구현을 수정한다.

4. 랭킹 구현을 최소 수정한다.
   - 대상 파일: `ai/retriever.py`
   - 정렬 순서: task 우선순위, 지역 우선순위, 점수, id
   - task 우선순위는 명시적 의도에만 부여한다.
   - 넓은 단어는 false positive 테스트를 먼저 만든 뒤 보정한다.

5. 실제 DB probe를 실행한다.
   - `USE_OPENAI_LLM=0`, `USE_FAISS=0`으로 재현 가능한 fallback 경로를 확인한다.
   - 문화, 주거, 창업, 자산형성, 대출 프롬프트를 모두 확인한다.

6. 전체 검증을 실행한다.
   - `python -m compileall backend ai tests tools`
   - `python -m unittest discover -s tests`
   - `npm test`
   - `npm run build`

## 현재 acceptance 결과

| --- | --- | --- | --- |
| 문화 | 서울 사는 24세 청년인데 문화생활 지원 정책 없나? | 봉사/건강보다 문화생활 core 신호 우선 | 신진작가, 관광코스가 상위 |
| 주거 | 서울 성북구 24세 청년 월세 지원 | 단순 주택 공고보다 월세 대출/지원 우선 | 월세자금보증, 청년맞춤형 월세대출 상위 |
| 창업 | 서울 29세 예비창업자 창업 지원금 | startup/policy_startup 우선 | 서울 창업 프로그램 상위 |
| 자산 | 서울 24세 목돈 마련 | 교육/대출보다 저축계좌/자산형성 우선 | 청년내일저축계좌, 청년도약계좌 상위 |
| 대출 | 서울 24세 생활비 대출 | 일반 금융보다 loan 우선 | 학자금 생활비 대출, 대출 상품 상위 |

## 회귀 테스트로 고정한 리스크

- `생활비`를 `복지`로 오분류하지 않는다.
- `문화생활`을 `복지`로 오분류하지 않는다.
- `전공연계` 안의 `공연`을 문화 공연으로 오인하지 않는다.
- `문화교류 봉사`가 문화 행사/관람비보다 앞서지 않는다.
- `저축` 단어가 들어간 재테크 교육이 저축계좌/적금/매칭 정책보다 앞서지 않는다.
- 월세 지원 의도에서 지역 주택 매물성 항목이 월세 대출/보증 항목보다 앞서지 않는다.
